Textarea

The .textarea class provides a styled multi-line text input with support for labels, validation states, and various sizing options.

Basic Examples

<textarea class="textarea" rows="3" placeholder="Enter text..."></textarea>
<textarea class="textarea" required></textarea>
<textarea class="textarea" disabled>Disabled content</textarea>
<textarea class="textarea" readonly>Read-only content</textarea>

Sizes

<textarea class="textarea form-sm" rows="2"></textarea>
<textarea class="textarea" rows="3"></textarea>
<textarea class="textarea form-lg" rows="4"></textarea>

Validation States

Please enter a valid description (min 20 characters).
Looks great! 
<textarea class="textarea input-error"></textarea>
<span class="form-feedback form-feedback-error">Error message</span>

<textarea class="textarea input-valid"></textarea>
<span class="form-feedback form-feedback-valid">Looks great!</span>

Auto-resize

<textarea
  class="textarea"
  @input="$el.style.height = 'auto'; $el.style.height = $el.scrollHeight + 'px'"
  style="overflow:hidden; resize:none"
></textarea>

Best Practices

  • Always provide a meaningful <label> associated with the textarea.
  • Use placeholder text to hint at expected content, but don't rely on it as a substitute for a label.
  • Choose an appropriate initial rows value based on expected content length.
  • Use the auto-resize pattern for open-ended content like comments or descriptions.
  • Show validation feedback immediately after the user leaves the field (@blur).
  • Use disabled only when input must be prevented; use readonly when the value should be visible but not editable.
